================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: "Unknown column 'created_by'" error still appears
SOLUTION:
   1. Verify migration completed (green success message)
   2. Run in PhpMyAdmin: DESCRIBE user_ledgers;
   3. If column missing, manually run:
      ALTER TABLE user_ledgers ADD COLUMN created_by INT NULL;

PROBLEM: Migration won't load or shows error
SOLUTION:
   1. Check database credentials in: app/config/database.php
   2. Verify connection details match your hosting provider
   3. Ensure database user has:
      - CREATE TABLE permission
      - ALTER TABLE permission
      - SELECT/INSERT/UPDATE permission
   4. Check error log: storage/logs/php-errors.log
   5. If credentials wrong, update and try again

PROBLEM: Approval succeeds but no ledger entry appears
SOLUTION:
   1. Check storage/logs/php-errors.log for errors
   2. Verify column exists: DESCRIBE user_ledgers;
   3. Check if ledger_synced flag is blocking it:
      SELECT ledger_synced FROM expenses WHERE id=70;
   4. Clear any PHP cache (if applicable)

PROBLEM: Migration shows "Column 'created_by' already exists" error
SOLUTION:
   This is NOT an error - it just means the column already exists
   This is actually good - means nothing broke
   The migration just skipped adding it again
